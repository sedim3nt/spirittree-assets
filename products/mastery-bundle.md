The OpenClaw Mastery Bundle is a set of ten comprehensive guides covering every layer of a production AI operations setup. Each guide is 1,000–3,000 words of practical, command-level instruction — not theory, not prompts, not fluff.

---

## The Learning Path

The guides are designed to be read in sequence, but each works as a standalone reference. Start at your current level and work forward.

### Stage 1: Foundation (Weeks 1–2)

**Start here if:** You're new to OpenClaw or setting up for the first time.

---

#### 📘 Quick-Start Guide
*From zero to first conversation in 30 minutes*

12 steps covering everything from `brew install openclaw` to your first cron. Includes the SOUL.md template, MEMORY.md setup, Telegram bot configuration, and a phased roadmap for your first 3 months. The essential first read.

**Best for:** New users, people who prefer hands-on learning over documentation

---

#### 🔒 Security Checklist
*10-point audit for personal AI operations*

A complete security audit with real commands for each check. Covers file permissions, skill review, gateway config, memory access, prompt injection testing, network exposure, cron auditing, sub-agent permissions, secret scanning, and SkillScan. Includes fix-it instructions for every finding.

**Best for:** Anyone who's deployed OpenClaw and wants to verify their setup is solid

---

### Stage 2: Optimization (Months 1–2)

**Start here if:** You have OpenClaw running and want to improve your setup.

---

#### 💰 Model Guide
*Choosing the right AI model for every task*

Complete comparison matrix for Claude Opus 4.6, Claude Sonnet 4.6, Claude Haiku 3.5, GPT-5.4, and Gemini 2.5 Pro. Task routing rules, flat-rate vs. API cost analysis, a monthly cost calculator template, and real production examples. Stops the most common waste: using expensive models for simple tasks.

**Best for:** Anyone spending more than $20/month on API costs

---

#### 💬 Telegram Handbook
*Complete setup guide for multi-topic agent operations*

Everything from creating a bot to running a fully organized forum with six dedicated topics. Covers group setup, bot permissions, OpenClaw JSON config, topic organization, cron delivery routing, BULLETIN.md cross-topic coordination, migration from single DM, and model-per-topic strategy.

**Best for:** Anyone still using CLI or direct DM — Forum Topics is significantly better for organized operations

---

#### 🛒 Buying Guide
*10-criteria checklist for evaluating AI skills and toolkits*

Before you spend money on any OpenClaw product, skill, or toolkit, use this checklist. Covers the "Is this just a prompt?" litmus test, security red flags, maintenance signals, community review interpretation, and the SkillScan procedure for auditing any installed skill.

**Best for:** Anyone evaluating third-party skills, plugins, or AI toolkits

---

### Stage 3: Scale (Months 2–3)

**Start here if:** You have a running system and want to expand what it does.

---

#### 📬 Substack Publishing System
*API-driven drafting, publishing, and cross-platform distribution*

Complete Substack API guide: SID cookie extraction, draft creation with HTML formatting rules, publishing with scheduling, byline configuration, cross-posting scripts for X/Facebook/Bluesky, editorial calendar template, cover image generation, quality checklist, and growth tactics.

**Best for:** Anyone publishing a Substack newsletter who wants to automate the pipeline

---

#### 🐙 Open Source Contribution System
*From issue discovery to merged PR — an agent-assisted workflow*

Systematic approach to open source contribution using the `gh` CLI. Covers issue filtering, repository evaluation framework (7 criteria), PR generation workflow, documentation improvement patterns, commit message conventions, maintainer etiquette, contribution tracking, and a reputation-building strategy.

**Best for:** Developers who want to build reputation and leverage through open source

---

#### 🎯 Client Playbook
*A complete consulting delivery system for AI operations*

Everything you need to run professional AI consulting engagements: pre-call intake questionnaire, 60-minute session facilitation guide, written roadmap template, handoff walkthrough script, 14-day bug fix warranty, monthly retainer structure, pricing rationale, Calendly booking flow, post-engagement email sequence, SOW template, and client feedback form.

**Best for:** Anyone considering or already running AI consulting work

---

### Stage 4: Mastery (Month 3+)

**Start here if:** You have a working, deployed system and want to optimize at the architectural level.

---

#### 🏗️ Stack Assessment
*9-layer framework for auditing any AI operations setup*

Complete framework covering L0 Substrate through L8 Commons. For each layer: what it is, key question, what most clients have, what they're missing, coverage scoring rubric (0–4), and priority ranking guide. Includes gap analysis template, 104 primitives reference list, and remediation roadmap template. The tool for diagnosing why a setup isn't delivering.

**Best for:** Anyone doing consulting, or anyone who wants a comprehensive audit of their own setup

---

#### 🌐 Frontier Operations
*Staying current at the edge of human-agent work*

Five skills for operating at the frontier: Boundary Sensing (monthly checklist), Seam Design (human/agent handoff templates), Failure Modeling (FAILURE_MODEL.md template with categories), Capability Forecasting (quarterly assessment worksheet), and Leverage Calibration (delegation decision matrix). Monthly review protocol. Real examples from 3+ months of production.

**Best for:** Anyone running a serious multi-agent operation who wants to stay ahead of the curve

---

#### 🧠 Intent Engineering
*Teaching agents what you actually want*

The complete framework: value hierarchy encoding, four trade-off matrices (speed/quality, cost/thoroughness, growth/sustainability, autonomy/safety), decision boundary definitions, 5-tier autonomy ladder, and the 4-discipline prompting stack. Real AGENTS.md before/after comparison. The difference between an agent that follows instructions and one that understands your intent.

**Best for:** Advanced users who want more sophisticated agent behavior — especially useful before setting up multi-agent operations
